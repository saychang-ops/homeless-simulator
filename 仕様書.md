# Homeless Simulator 仕様書 v6.0

## コンセプト
- 東京・新宿を舞台にした超現実的なサバイバル・シミュレーター。
- UI：1980年代コマンド選択式アドベンチャー風。CRTモニター走査線エフェクト。
- チープな8ビット風ピクセルアート。ひらがな中心のテキスト。
- **すべてリアルタイム連動**。ブラウザを閉じても現実と同じように時間が経過し、状態も変化する。

## ゲームシステム

### 1. リアルタイム減少
- 現実時間と完全連動。放置・ブラウザ閉鎖中も状態が変動する（lastTickTimeで差分計算）。
- 睡眠中はHP回復・空腹進行・えいせい微減・病気ダメージのみ。
- 作業中はリアルタイムで進捗を管理。

### 2. 体感温度
```
たいかんおんど = リアル気温 + 天候補正 + 拠点ボーナス(公園のみ) + 衣服ボーナス合計 + しんぶんし(+1℃)
```

| 天候 | 補正 |
|------|------|
| はれ | 0℃ |
| くもり | -2℃ |
| あめ | -6℃ |
| ゆき | -12℃ |
| きょうふう | -4℃ |

### 3. 病気システム

| 病気 | 発症条件 | HP減少（Lv1〜5） | 悪化間隔 |
|------|---------|----------------|---------|
| かぜ | 体感温度≤5℃ かつ 体温<35℃（確率10%/h） | 3〜20/h | 24h |
| ていけっとう | 空腹0が2時間継続 | 5〜30/h | 24h |
| ひふしっかん | えいせい≤30（確率10%/h） | 1〜12/h | 24h |

- **治療**: 薬を使う、医療支援イベントに参加（Lv-2）、病院（そうだんまどぐち）で処方（薬をもちものに追加）
- **ひふしっかんLv2以上**: 会話拒否・日雇い不可
- **病気の自然回復**: かぜは体感温度≥15℃かつ体温≥36℃で確率5%/hで回復

### 4. におい（えいせい）
- えいせいが25未満になると「におい」状態になる。
- においがあると：会話の50%が失敗・rejectsSmellyのある店や場所に入れない。
- 銭湯（¥500）でえいせいを大幅回復（+80）。

### 5. 衣服システム（8スロット）

| スロット | デフォルト | 購入可能品 |
|----------|-----------|-----------|
| アウター | なし | あつでジャンバー+10℃/¥3000（ふるぎや） |
| トップス | ジャージ(うわぎ)+2℃ | ニットセーター+4℃/¥1500（ふるぎや） |
| ボトムス | ジャージ(したぎ)+2℃ | チノパン+3℃/¥800（ふるぎや） |
| インナー | Tシャツ+1℃ | ヒートテック+4℃/¥800（ふるぎや） |
| したぎ | トランクス+0℃ | - |
| ぼうし | なし | ニットぼう+2℃/¥400（ふるぎや） |
| マフラー | なし | マフラー+3℃/¥600（ふるぎや） |
| 雨よけ | なし | ビニールがさ+0℃/¥500（リサイクル）、かっぱ+2℃/¥2000（ふるぎや）、ブルーシート+0℃（もちものから使用） |

- 古着屋・リサイクルショップで購入するとその場で装備（もちものを経由しない）。
- 「しんぶんし」は専用管理（clothingスロット不使用）。使用後24時間で自動消滅。体感温度+1℃。
- 拠点Lv1以上で「アイテムボックス」が開放。衣服をしまう／とりだすことが可能。

### 6. 雨天ペナルティと雨具

- 天候が「あめ」「きょうふう」「ゆき」の際、**雨具を装備していない**場合は「濡れ」状態になる。
  - **たいかんおんど**: -3℃の追加ペナルティ
  - **えいせい減少速度**: 通常の倍（10/h）
- **対策**: 「雨よけ」スロットに以下のいずれかを装備することでペナルティを完全に防ぐ。
  - **ビニールがさ**（¥500・リサイクルショップ）
  - **かっぱ**（¥2,000・ふるぎや）
  - **ブルーシート**（もちものから「つかう」で装備）

### 7. 移動コスト

- 「ばしょいどう」を実行するたびに **たいりょく-2、くうふく-3** が消費される。無駄な移動を減らす戦略が重要になる。
- 銭湯などのサブエリア（店舗・施設）から外（親エリア）に出るだけの移動（「そとへでる」コマンド）に関しては、上記コストは免除される。

### 8. テキスト表示（タイピングエフェクト）

- 画面に表示されるすべての新規メッセージログは、左から右へ1文字ずつタイピングされるアニメーションで出力される。
- 各文字の表示に合わせて、当時のファミコン風格な短い電子音（矩形波ピピピ音）が鳴る（Web Audio APIで生成）。
- プレイヤーがテキスト出力中に画面をクリックすると、アニメーションをスキップし全文が一瞬で表示される。

## サブゲーム：ちんちろりん

- **開始条件**: 公園・「きょてん」メニューから。21:00〜翌2:59の間のみ。
- **BGM**: 専用BGMがループ再生（環境音に上書き）。
- **持ち金のセーブ**: `chinchiroYen` / `chinchiroDayKey` で管理。深夜3時区切りで同一ゲーム日内は持ち金が引き継がれる。翌3時以降リセット。
- **1日1回制限**: 先輩全員が破産して終了した場合、その日は再開不可。

### ゲーム進行
1. **親決め**: 全員が1個サイコロを振り、最大値のプレイヤーが「親」。
2. **ベット**: 子が賭け金（¥100〜全額）を設定。
3. **振り（ロール）**: 親→子の順で1人最大3回。役が確定した時点で手番終了。
4. **清算**: 親の即勝ち/即負けは全体一括。それ以外は子と個別対決。

### 役一覧
| 役 | 条件 | 倍率 | 即決 |
|----|------|------|------|
| ピンゾロ | 1・1・1 | 5倍 | 即勝ち |
| アラシ | ゾロ目（1以外） | 3倍 | 即勝ち |
| シゴロ | 4・5・6 | 2倍 | 即勝ち |
| 通常の目 | 2つ揃い（残り1つが「目」） | 1倍 | 目の大小で勝負 |
| 1のめ | 2つ揃い・残り1 | 1倍 | 即負け |
| メナシ | 3回振っても役なし | 1倍 | 即負け |
| ションベン | サイコロこぼれ（5%） | 1倍 | 即負け |
| ヒフミ | 1・2・3 | 2倍 | 即負け |

### 精算とログ形式
- **ログの1行集約**: 精算時のメッセージは `名前「セリフ」 ¥金額` の形式で1行にまとめて表示する。
- **表示制限**: ログ表示欄は最新の3行のみを表示。4行目以降は自動で最下部へスクロール・追従する。
- **インジケーター**: 上下に隠れたログがある場合は、コマンド欄と同様の点滅▲▼マーカーを表示する。

## 環境音システム

- 各エリア×時間帯×天候に応じた環境音がループ再生。
- エリア移動・天候変化時はクロスフェードで切り替わる。
- 睡眠中・作業中は専用の環境音。
- ちんちろりん中は専用BGMに切り替わる（通常環境音は停止）。
- ブラウザの自動再生制限のため、初回クリックで有効化。

### 店舗専用BGM（8bitシンセ・アンサンブル）

- 5つの店舗サブエリアに入ると、Web Audio APIでリアルタイム合成される8bitシンセBGMが自動再生される。
- **音響システム**: 単音ループから進化し、主旋律（矩形波）、ベース（三角波）、パーカッション（ノイズ）の最大3チャネル＋エフェクト（Delay/Filter）で構成。
- **新宿の哀愁と喧騒**: 各店舗ごとに、新宿という街の情緒を反映した旋律と質感を実装。

| 店舗 | 波形・チャネル | 楽曲コンセプト | 特殊効果 |
|------|--------------|--------------|---------|
| せんとう | 三角波(ベース) + 矩形波 | 郷愁と安らぎ（ヨナ抜き音階） | Delay (エコー) |
| べんとうや | 矩形波(高速) | 庶民の喧騒（事務的・単調） | - |
| ふるぎや | 三角波 + 矩形波 | 都会のノスタルジー（ジャジー） | LFO (ヨレ・ビブラート) |
| リサイクル | のこぎり波 + ノイズ | 吹き溜まりのジャンク（不安定） | BiquadFilter |
| ふくびきじょ | 矩形波(高音) + ノイズ | 空虚な祝祭（お祭り騒ぎの裏側） | 高速アルペジオ |

- 各音色はADSR（エンベロープ）制御により、アタックとリリースの質感を最適化。
- 店舗を出ると即座にシンセBGMが停止し、通常環境音に戻る。

## UI・レイアウト

### 全体構造
- **ゲーム画面コンテナ**: `width: 100vw; height: 100dvh` の全画面ベースに、アスペクト比 **16:9** で拘束されたゲームエリアをセンタリング表示（レターボックス）。
  - 幅: `min(100vw, 100dvh × 16/9)`
  - 高: `min(100dvh, 100vw × 9/16)`
- **レイアウト方針**: フッター（コピーライト等）を廃止し、すべての要素を垂直方向に詰めて配置。スマホ画面での見切れを防止する。
- **URLバー復旧ボタン**: ブラウザのURLバーが不意に表示された際、画面**左上**に赤い点滅の `▲` ボタンを表示。タップで再フルスクリーン化を試行。

### 2カラム構成（flex比率 **7:3**）

| カラム | 比率 | 内容 |
|--------|------|------|
| 左カラム | **7** | 背景画像（**16:9**）＋ ログエリア |
| 右カラム | **3** | 日時・天気（13px）＋ コマンド群 |

- **背景画像**: `aspect-ratio: 16/9`、`object-fit: cover` で横幅いっぱいに表示（4:3画像は上下をわずかにクロップ）。
- **右カラム日時ヘッダー**:
  - 1行目: `月/日　HH:MM`（日付・時刻）
  - 2行目: `天気の説明　XX℃`（天気・気温）
  - フォントサイズ: 13px
- **コマンドボタン**: `whiteSpace: normal; wordBreak: break-all` でラベルが長い場合は自動折り返し。
- **スクロールインジケーター**: コマンドリストが下に続く場合、エリア最下部に `▼`（点滅）を表示。最下部到達で自動消灯。
- ちんちろりん中は左カラムの画像エリアがサイコロボード（16:9）に切り替わる。レイアウト構造は同一。
- CRTスキャンライン・ビネットエフェクトをオーバーレイ。

## スマートフォン対応

### ビューポート設定（index.html）
- `user-scalable=no` / `viewport-fit=cover` でピンチズーム・セーフエリア対応。
- PWA対応メタタグ（`mobile-web-app-capable` / `apple-mobile-web-app-capable`）。

### CSS対応（index.css）
- `height: 100dvh`（Dynamic Viewport Height）でモバイルブラウザのアドレスバーによる見切れを防止。
- `touch-action: none` / `overscroll-behavior: none` でスクロール・スワイプ誤操作を防止。

### 横画面強制（App.jsx）
- **タイトル画面マウント時**（表示直後）に即座に `requestFullscreen()` を試行（URLバー早期消去）。
- **TOUCH TO START タップ時**にも `requestFullscreen()` + `screen.orientation.lock('landscape')` を実行（二重保険）。
- ブラウザ非対応環境では例外を無視（`.catch(() => {})`）。

### 縦向き警告オーバーレイ
- CSS `@media screen and (orientation: portrait)` で縦向き検出。
- 縦持ちの場合は「↻ スマホを横にしてください」の全画面オーバーレイを自動表示（z-index: 9999）。
- 横に戻すと自動で消える。タイトル画面・ゲーム本編の両方に適用。
