# 新しい仕様のアイデア17（スマホ全画面・横画面対応）

## 📌 概要
スマホ（実機）のブラウザで表示した際に、はみ出し（見切れ）が発生してしまう問題を修正し、どんなスマホでも画面いっぱいに（かつ比率を崩さずに）綺麗に表示されるようにします。
また、ブラウザのURLバーを消すための**フルスクリーン化**および、モバイルでの**横画面（ランドスケープ）強制**のアプローチを導入します。

## 💡 仕様と実装アイデア

### 1. `dvh` (Dynamic Viewport Height) の導入
現在のCSSでは、画面の高さを取るために `100vh` などを使っている可能性がありますが、これだとスマホのURLバーが表示された状態ではみ出してしまいます。
モダンCSSである `100dvh` を使用することで、URLバーの増減に合わせて正確に画面内に収まるようになります。

### 2. Fullscreen API の活用（URLバーを消す）
「TOUCH TO START」をクリックしたタイミングは、**ブラウザのフルスクリーン化権限（Fullscreen API）を取得できる完璧なタイミング**です。
BGMの再生開始と同時に、JS側で `document.documentElement.requestFullscreen()` を実行し、ブラウザのURLバーなどのUIをすべて消し飛ばして完全な全画面にします。

### 3. 横画面のロック表示（Screen Orientation API）
同じくスタートボタンを押した際に、強制的にスマホを横画面（ランドスケープ）にする `screen.orientation.lock("landscape")` を実行します（※AndroidのChromeなどで有効です。iPhoneの場合はOSの仕様上ロックできないことがあるため、CSSで回転を促すか、自動縮小されるようにします）。

### 4. アスペクト比の固定と自動リサイズ
画面の比率がおかしくならないように、ゲーム全体のコンテナを `aspect-ratio` を用いて固定し、スマホの画面サイズに合わせて自動で拡大・縮小（レターボックス表示）されるようにCSSを調整します。

1. **index.html の viewport 設定更新**
   - 以下のmetaタグを追加/修正し、ユーザーのピンチズーム操作を無効化し、フルスクリーンアプリとして認識されやすくしてください。

   `<meta name="mobile-web-app-capable" content="yes">`
   `<meta name="apple-mobile-web-app-capable" content="yes">`
   `<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">`

2. **Full Screen / 横画面ロック API の実装と 各種スマホ (Android / iPhone) への対応 (App.jsx / index.css)**
   - タイトル画面の `TOUCH TO START` をクリック（またはタップ）したときの処理内に以下のロジックを追加してください。
     - BGM再生（既存の処理）
     - ブラウザのフルスクリーンを実行：`if (document.documentElement.requestFullscreen) { document.documentElement.requestFullscreen().catch(e => console.log(e)) }`
     - 画面を横向きにロック（Android用）：`if (screen.orientation && screen.orientation.lock) { screen.orientation.lock('landscape').catch(e => console.log(e)) }`
   - **【最重要: iPhone (iOS) / Android 両対応のためのフォールバック】** Androidでは上記のAPIで横画面＋フルスクリーンが正常に動作しますが、iOS (Safari等) はそれを正しくサポートしていません。そのため、iOSや縦持ち固定設定のAndroid端末対策として、CSSの `@media screen and (orientation: portrait)` を使用し、スマホが縦向き状態のときは画面全体（`.app-container`等）に対して `transform: rotate(-90deg);` をかけて**強制的に画面を横向きに描画**するか、もしくは「スマホを横にしてください」という全画面オーバーレイ（警告画面）を出して、横持ちでのプレイを促すフォールバックを必ず実装してください。

3. **CSSレイアウトの最適化（index.css）**
   - スマホのURLバーによる縦方向の見切れ（スクロールしてしまう問題）を防ぐため、`#root` や全体のコンテナの高さを `100vh` ではなく `100dvh` （Dynamic Viewport Height）に変更してください。
   - 画面がどの端末サイズでも崩れないように、ゲーム画面の親コンテナを `display: flex; justify-content: center; align-items: center; overflow: hidden;` にした上で、ゲームのメイン画面の比率（例: `aspect-ratio: 16 / 9` もしくは `800 / 600` 等）を保ちながら `max-width: 100vw; max-height: 100dvh;` でレターボックス状（余白は黒）にきれいに収まるレスポンシブなCSS設計にリファクタリングしてください。
   - スクロールやスワイプでの戻る動作が一切起きないようにしてください：`body { overflow: hidden; touch-action: none; background: #000; overscroll-behavior: none; }`
