# 新しい仕様のアイデア11：UIレイアウトの改修（ログ・コメント欄の配置変更）（実装済）

## 概要
ユーザーからのフィードバックに基づき、画面下部のログ（コメント）欄のレイアウトを調整します。ポートピアのような画面構成を目指しつつ、画像とテキストを分離する配置案です。

## 対象となる要望
「ゲーム全体のレイアウトについて、下部のコメント欄は画像の下端の１行目から開始するようにして、右側のコマンドの境界手前で文章を改行するようにしてください。」
「画像とコメントは絶対にかさなりません！」

## 変更内容詳細（実装済）

1. **日付・時刻・気温・天気の表示復活**:
   - 右パネル上部（コマンド群の上）に常時表示を復元。
   - 1行目: `月/日　HH:MM`
   - 2行目: `天気の説明　XX℃`

2. **画像との非干渉（オーバーラップ禁止）**:
   - 画像表示エリアとテキスト表示（コメント・ログ）エリアは完全に分かれたブロック。
   - **絶対に画像と文字が重ならない**。

3. **テキストの開始位置（1行分の空きスペース）**:
   - ログエリアの上部に「1行分のスペース（`marginTop: FS = 18px`）」を確保。
   - 画像の下端から1行下がったところからコメント・ログの1行目が始まる。

4. **ログエリアの幅制限（コマンドエリアとの分離）**:
   - ログエリアの横幅を画像エリアと同じ幅（約58%）に制限。
   - 右側のコマンドエリアにテキストが侵入しない。

5. **レイアウト構造の刷新**:
   - **変更前**: 上部（画像＋コマンドの横並び）＋ 下部（ログ独立セクション）の2段構成。コマンドが縦に伸びるとログ開始位置が下にずれる問題があった。
   - **変更後**: **左カラム（58%固定）に画像＋ログ、右カラムに日時・天気・コマンド**の2カラム構成に統合。ログ開始位置がコマンドの高さに左右されなくなった。

6. **ちんちろりん画面への適用**:
   - ちんちろりん実行中（サイコロ画面）も同じ2カラム構造を維持。

## 実装概要（App.jsx）
- 外側のflexコンテナを `flex: 1` に変更し、全高を占有するよう調整。
- 左カラム: `width: '58%', flexShrink: 0, display: 'flex', flexDirection: 'column'`
  - 画像（またはちんちろりんボード）を上部に配置。
  - `marginTop: FS` でログを画像直下に固定。
- 右カラム: `flex: 1` で残り幅を使用。日時・天気 → コマンドの順に縦並び。
- 旧来の独立した下部ログセクションは廃止。
