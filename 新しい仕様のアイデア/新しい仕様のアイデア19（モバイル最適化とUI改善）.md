# 新しい仕様のアイデア19：モバイル最適化とUI改善

## 1. モバイル・フルスクリーン体験の刷新
- **フルスクリーンの強制**: 起動時に `requestFullscreen()` を呼び出し、URLバーがない状態を標準化する。
- **URLバー自動復旧ボタン**: ブラウザのURLバーが誤って表示されたことを検知し、画面**左上**に赤い点滅の `▲` を表示。タップすると再フルスクリーン化を試行してボタンを消す。
- **オープニング画像の迫力向上**: `title.png` の表示サイズを拡大し、画面いっぱいに広がるよう調整。

## 2. メインUIのレイアウト最適化
- **フッター要素の全削除**: 画面下部のコピーライト等のテキストを完全に削除。
- **UIの垂直位置調整**: 空いたスペースを活用して画面全体の垂直方向の間隔を詰め、スマホ画面で下部が切れないように表示を底上げする。

## 3. ちんちろりん・ログ表示の改善
- **ログの1行集約**: 精算時のメッセージを `名前「セリフ」 ¥金額` の形式に統合し、可読性と省スペース化を両立。
- **コメント欄の3行制限と自動スクロール**: 最新の3行のみを表示。4行目以降は自動で最下行へスクロールし、古い行を隠す。
- **スクロールインジケーター**: 上下に隠れたログがある場合、点滅する `▲` または `▼` マーカーを表示。

## 4. 所持金同期バグの修正
- **概要**: 拠点でちんちろりんを開始する際、最新の所持金が反映されず0になる現象を修正。
- **対策**: ゲーム本編からミニゲームへの遷移時に、常に `status.yen` を最新の参照として渡す。
