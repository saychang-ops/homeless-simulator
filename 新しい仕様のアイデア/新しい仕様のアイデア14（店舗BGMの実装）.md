# 新しい仕様のアイデア14（店舗BGMの実装）

## 概要
銭湯、弁当屋、古着屋、リサイクルショップ、福引所の各サブエリア（店舗・施設）に入った際、それぞれの雰囲気に合った専用のBGMを再生する。
**実装手法**: 外部の音声ファイル（mp3など）は一切使用せず、ブラウザ標準の **Web Audio API（OscillatorNode）** を用いて、プログラム側で直接「ファミコン風の8bitチップチューン」をリアルタイム生成・演奏する自作シーケンサーを実装する。

## なぜこの手法（対策案B）が良いのか？
- **最高にレトロな音質**: 生成AIが作ったMP3よりも、Web Audio APIの `square`（矩形波）や `triangle`（三角波）を直接鳴らす方がノイズが全くなく、**完璧に当時（80年代ファミコン時代）のピュアなピコピコ音**を再現できます。
- **ファイルサイズゼロ**: 音声ファイルを読み込まないため、ロード時間が一切かかりません。
- **タイピング音との親和性**: 先に実装したタイピングのSE（ピッピッ）と同じ仕組みで音が鳴るため、音源の質感が完全に一致し、世界観が統一されます。

## 実装の要件
1. **`soundManager.js` の改修とシーケンサーの実装**
   - 音階の周波数（周波数テーブル）と、各店舗用の簡単なメロディ配列（休符を含むループ用の音符データ）を定義する。
   - `setInterval` や Web Audio APIのスケジューリング機能を用いて、配列のデータを順に読み取り `OscillatorNode` で演奏し続ける（ループ再生する）メソッドを追加。
   - 店舗のジャンルに合わせ、以下のような短いループメロディ（数小節程度）を各店舗分プログラミングする。
     - ♨️ **銭湯 (`sento`)**: ゆっくりとした四分音符中心の和風メロディ（ヨナ抜き音階など）。
     - 🍱 **弁当屋 (`bentoya`)**: テンポが良く明るい、アイテム屋風のメロディ。
     - 👕 **古着屋 (`furugiya`)**: 少しハネたリズムや、おしゃれな和音を感じさせるメロディ。
     - ♻️ **リサイクルショップ (`recycle`)**: 半音階やマイナー・スケールを混ぜた、少し怪しい雰囲気のメロディ。
     - 🎯 **福引所 (`fukubiki`)**: 短くループするアップテンポなファンファーレ風メロディ。
2. **既存サウンドとの切り替えロジック**
   - 店舗（`parentArea` が設定されているエリア）に入った際は、現在再生中の環境音（MP3）を停止し、Web Audio APIによるBGM演奏を開始する。
   - 外に出た際は、シンセBGMを停止し、元のMP3環境音へ戻す。

1. **音階データとメロディの定義（`soundManager.js` 上部）**
以下の周波数テーブルと、各店舗用のメロディデータ（BPM、波形、音符配列）を定数として追加してください。音符配列は `[音階名, 16分音符を1とした長さ]` です。

```javascript
const FREQ = {
    C4: 261.63, C_hash4: 277.18, D4: 293.66, Eb4: 311.13, E4: 329.63, F4: 349.23, F_hash4: 369.99, G4: 392.00, G_hash4: 415.30, A4: 440.00, Bb4: 466.16, B4: 493.88,

2. **8bitシーケンサーロジックの実装（`SoundManager` クラス内）**
   - `playSynthBGM(areaId)` と `stopSynthBGM()` メソッドを実装してください。
   - `SHOP_MELODIES[areaId]` を取得し、なければリターン。
   - `this.current` (MP3環境音) が鳴っていれば `this.stop()` などで止めてください。
   - `this.audioCtx` を用いて、`setTimeout`（またはWeb Audio APIの正確なスケジュール）によりループ再生を行うシーケンサーを構築します。
   - `16分音符のミリ秒数 = (60 / tempo) / 4 * 1000` で計算し、各ノートの長さを乗算して次の音の再生タイミングを `setTimeout` で回すのが手軽かつレトロな揺らぎが出て良いです。
   - 発音時は都度 `OscillatorNode` と `GainNode` を作成し接続。音量（gain）は主環境音の邪魔にならないよう `0.05` 程度に控えめにし、音と音の間に僅かな隙間（スタッカート・アタック感）を作るとよりピコピコ感が出ます（発音時間を指定時間より20msほど短くして `stop` する）。
   - 音程が `rest` の場合は何も発音せず `setTimeout` のみ回します。

3. **再生の切り替え連動（`update` メソッドの修正等）**
   - `update` 関数内でエリア (`area`) を判定する際、`SHOP_MELODIES` に存在するキーなら、通常のMP3環境音（`SOUND_MAP`）の処理へ進まずに新たに `playSynthBGM(area)` を呼び出すように変更してください。
   - 外に出た（店舗エリア以外になった）際は `stopSynthBGM()` を呼び出し、元の処理（MP3の環境音の最新化・クロスフェード）へと戻るように制御（フラグ管理等）してください。
```
